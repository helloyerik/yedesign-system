# yedesign-system

Независимая дизайн-система на `Vue`, вынесенная в корень `vibecoding`, чтобы новые продукты и внутренние сервисы не приходилось собирать с нуля.

Это не отдельный “дизайн для одного проекта”, а общая база для разных продуктов:
- единые токены;
- общие UX-паттерны;
- повторно используемые компоненты;
- Storybook как живая документация;
- локальные `guidelines` как слой правил проектирования.

## Зачем это нужно

Цель системы:
- ускорять старт новых проектов;
- уменьшать количество кастомных решений;
- повышать консистентность интерфейсов;
- снижать стоимость поддержки;
- уменьшать количество правок на ревью и на этапе реализации;
- переносить удачные паттерны между разными продуктами.

## Что использовать как основу

При создании нового проекта внутри `vibecoding`:
- сначала смотреть в [`../guidelines`](../guidelines/README.md);
- затем смотреть, есть ли нужный паттерн в `yedesign-system`;
- только потом делать новый компонент или новый сценарий.

## Структура

- `packages/tokens`
  source of truth для токенов
- `packages/components`
  библиотека Vue-компонентов
- `apps/storybook`
  документация, примеры и foundations

## Тема и вариативность

Система должна поддерживать разные продукты без пересборки UX с нуля.

Практически это значит:
- общая логика компонентов и паттернов остается одной;
- меняются брендовые токены;
- меняются радиусы;
- меняются шрифты;
- при необходимости меняются отдельные surface- и semantic-настройки.

То есть подход такой же по идее, как у систем с theme-layer:
- одна база UX и компонентов;
- много комбинаций визуальной темы поверх нее.

## Что можно тематизировать

Минимальный слой темы:
- brand color;
- hover / active цвета;
- radius scale;
- font family;
- базовые surface и text colors.

Не надо тематизировать без причины:
- расстояния между полями;
- порядок действий;
- логику форм;
- валидацию;
- базовые UX-паттерны.

Визуальный бренд может меняться, а поведение системы должно оставаться предсказуемым.

## Использование как зависимости

Система теперь собирается как потребляемый пакет:
- `@yerik/yedesign-system`

Минимальный сценарий подключения во Vue-проекте:

```js
import { createApp } from "vue";
import App from "./App.vue";
import "@yerik/yedesign-system/tokens.css";
import "@yerik/yedesign-system/style.css";

createApp(App).mount("#app");
```

Импорт компонентов:

```js
import { Button, Dialog, ListItem, RadioGroup, Tag } from "@yerik/yedesign-system";
```

Локально внутри `vibecoding` это можно подключать через `file:`-зависимость.
Для внешнего потребления без дублей следующий шаг — вынести `yedesign-system` в отдельный git-репозиторий или registry и заменить `file:` на git-url или package registry.

## Как использовать вместе с guidelines

`guidelines` отвечают на вопрос:
- как проектировать;
- когда использовать паттерн;
- зачем он нужен;
- какую пользу он дает бизнесу и пользователю.

`yedesign-system` отвечает на вопрос:
- чем это собирать;
- какие токены и компоненты использовать;
- как переиспользовать уже готовое решение.
